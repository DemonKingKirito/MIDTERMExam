    • Automatic Communications with Clients and Customers

Benefits

  1. Reduces errors
    2. Monotony can lead to complacency, but automation helps avoid mistakes.

  2. Saves time
    3. Automating repetitive tasks frees up work time.

  3. Enhanced security
    4. Self-detection, self-healing, and auto-correction of security threats.

Challenges

  1. Vendor interoperability, or managing how supplier equipment works together
  2. Data and configuration consistency such as standards and APIs
  3. Resources to implement network automation, notably people and money

Cloud Provisioning

Types

  1. Advanced Provisioning
    2. A formal agreement between the user and the cloud service provider that specifies the type and quantity of cloud resources that are required

  2. Dynamic Provisioning
    3. The cloud service provider adds or removes resources as needed based on the demand or workload of the applications

  3. User Self-Provisioning
    4. The user can request and access cloud resources on demand without the intervention of the cloud service provider

benefits

  1. Faster delivery
    2. Cloud provisioning can bring all your cloud data and resources together to improve productivity and IT visibility, and deliver cloud services faster

  2. Cost savings
    3. When used correctly, cloud provisioning allows organizations to spend less on managing and maintaining IT systems

  3. Automation
    4. Cloud provisioning can automate cloud management by using policies to govern deployments and resource operations

Challenges

  1. Complex management and monitoring.
  2. Resource and service dependencies.
  3. Policy enforcement.
  4. Cost controls
  5. Vendor lock-in
  6. Compliance and legal issues
  7. Specialized technical expertise.
